01Attention Is All You Need鸟瞰 · Overview

一座由相同积木堆起的塔

2017 年那篇《Attention Is All You Need》最颠覆的一点，是彻底抛弃了 RNN 一个词等一个词的串行计算——整句话被同时喂进去，所有位置并行处理。这让模型能吃下海量数据，训练得又深又快。

别被论文里的方块图吓到。一个 decoder-only 的 Transformer（GPT 这一系）本质上只做三件事：把词查成向量（Embedding）、让向量在一摞结构完全相同的 Block 里反复被改写、最后把向量翻译回一个词的概率分布。这一摞 Block——GPT-2 有 12 层、GPT-3 有 96 层——每一层都长得一模一样。

全塔鸟瞰 / Anatomy
整篇文章，我们就跟着一句 the cat sat 的向量，从塔底走到塔顶。
02Token Embedding入口 · Lookup

先把词查成一串数字

计算机不认识 "cat"，只认识数字。第一步，分词器（tokenizer）把句子切成 token，每个 token 是词表里的一个整数 id——the cat sat 在 GPT-2 的 BPE 词表里大致对应 1169 / 3797 / 3332 三个 id。

紧接着是一张巨大的查找表：嵌入矩阵（embedding matrix），形状 50257 × 768——词表里每个 token 都对应一行 768 个数。所谓"嵌入"，就是拿 id 去这张表里取出那一行。"cat" 不再是 3797，而是一个 768 维向量，它在向量空间里的位置编码了"猫"的含义。

token → 768 维向量 / Lookup
这 768 个数一开始是随机的，是训练把语义"挤"进了这个空间——意思相近的词，向量也相近（king − man + woman ≈ queen 的那种相近）。
03Positional Encoding顺序 · Order

但顺序去哪了？

这里藏着 Transformer 的一个先天缺陷：注意力本身分不清词序。把 "the cat sat" 打乱成 "sat the cat"，若只看词向量的集合，注意力算出来的结果一模一样——它是"置换不变"（permutation-invariant）的。可语言里，顺序就是意义。

解法是给每个位置加一个位置编码（positional encoding）向量，直接加到词向量上。原版论文用一组不同频率的正弦/余弦波，低维慢、高维快，让每个位置都有独一无二的"指纹"：

\[ PE_{(pos,\,2i)}=\sin\!\Big(\tfrac{pos}{10000^{2i/d}}\Big),\qquad PE_{(pos,\,2i+1)}=\cos\!\Big(\tfrac{pos}{10000^{2i/d}}\Big) \]
不同频率的正弦波 / Sinusoids
最终送进塔的，是 词向量 + 位置向量。GPT 改用可学习的位置向量；如今主流是 RoPE（旋转位置编码），把位置旋转进 Q、K 里，对相对距离更友好、外推更稳。
04Residual Stream核心心智模型

一条主干道，子层只读写它

这是看懂 Transformer 最关键的一张图。把那个 768 维向量想成一条贯穿到塔顶的主干道（residual stream）。每个子层都不直接替换它，而是从主干道上读当前的 x，算出一个"增量"，再加回主干道：

\[ x \;\leftarrow\; x + \operatorname{Sublayer}(x) \]

向量一路上只被逐步修订，从不推倒重来。一个 Transformer Block 里有两个这样的子层串在主干道上：先是多头注意力（让不同位置交换信息），再是前馈网络（逐位置加工）。每个子层都包着"读 → 算 → 加回"这同一个壳。

残差流主干道 / The Highway
05Layer Normalization稳定 · Stabilize

每次动手前，先把数字捋平

向量在主干道上被反复相加，数值容易越滚越大或忽高忽低，训练就会发散。所以每个子层在读取主干道之前，先做一次层归一化（Layer Normalization）：把这个向量的 768 个数减均值、除标准差，拉成均值 0、方差 1，再用两个可学习参数 γ、β 缩放平移：

\[ \operatorname{LN}(x)=\gamma\cdot\dfrac{x-\mu}{\sqrt{\sigma^2+\epsilon}}+\beta \]
归一化前 vs 后 / Centering
归一化前：参差、偏移μ = 0
注意它沿每个 token 自己的 768 维归一化（不是跨 batch），所以和句长、批大小都无关。现代模型几乎都用 Pre-LN（把归一化放在子层之前），比原版的 Post-LN 训练更稳。
06Multi-Head Attention核心 · Mixing

同一句话，十二种看法

注意力的内部——Q、K、V、缩放点积、softmax——在 《注意力机制，逐矩阵拆解》 已逐格算过；这里只看它在塔里的角色和"多头"的妙处。一个头把每个词投影成 Query、Key、Value，让每个位置去"看"句中其他位置、按相关度加权汇总信息。

\[ \operatorname{Attention}(\htmlClass{m-q}{Q},\htmlClass{m-k}{K},\htmlClass{m-v}{V})=\operatorname{softmax}\!\Big(\tfrac{\htmlClass{m-q}{Q}\htmlClass{m-k}{K^{\top}}}{\sqrt{d_k}}\Big)\htmlClass{m-v}{V} \]

但只用一个头太单薄。Transformer 把 768 维切成 12 个头、每头 64 维，并行跑 12 套独立注意力。不同头会自发学到不同关系：有的盯相邻词，有的连主谓，有的找指代。12 个头各算各的，再拼接（concat）回 768 维，过输出投影 W_O 融合，写回主干道。

同一句话上的若干注意力头 / Heads (示意)
行 = "谁在看"，列 = "看向谁"，颜色越亮注意力权重越大。12 个头里这是有代表性的 4 个：相邻、主谓、指代、全局。
07Residual Connection深度 · Why it trains

那个加号，才是深度的功臣

回到 x = x + Sublayer(x) 里的那个加号。它叫残差连接（residual / skip connection），看着不起眼，却是能把网络堆到几十上百层还训得动的关键。

原因在反向传播（见 《反向传播：梯度如何流动》）：梯度沿主干道倒流时，加号给了它一条恒等高速路——即便子层那一支把梯度衰减得几近归零，主干道这一支仍让梯度原样穿过，不会逐层相乘到消失。没有残差，12 层就已难训；有了它，96 层也畅通。

梯度逆流：无残差 vs 有残差 / Gradient flow
无残差 · 逐层衰减
有残差 · 畅通
每一格是梯度流过一层后剩下的强度，从塔顶往塔底读。左边乘着乘着就没了（梯度消失）；右边靠加号一路保住。
08Feed-Forward Network加工 · Knowledge

升维、激活、再降维

注意力让词与词交换信息；信息到手后，前馈网络（FFN，也叫 MLP）对每个 token 单独加工。它只有两层全连接，中间夹一个非线性激活：

\[ \operatorname{FFN}(x)=W_2\,\operatorname{GELU}(W_1 x + b_1)+b_2 \]

关键在中间维度：先把 768 维升到 4 倍、即 3072 维，过 GELU 激活，再压回 768 维。这个"先撑大、再收紧"赋予了模型非线性表达力。它也是参数大头——一个 Block 里 FFN 的参数约是注意力的两倍，许多事实性"知识"就存在这些权重里。

09Stacking N Layers深度 · Depth

把同一块积木，叠十二遍

一个 Block（注意力 + FFN，各带残差与 LN）就是一块积木。Transformer 把结构完全相同、但权重各不相同的 Block 摞起来：GPT-2 摞 12 层，GPT-3 摞 96 层。主干道上的向量穿过每一层，被多修订一次。

经验上，浅层更关注表层与语法（词形、相邻搭配），越往上越偏语义与抽象（指代、句意、事实）。同一个向量从塔底到塔顶，含义被一层层"提纯"——这就是"深度"带来的层级表征。

10Encoder · Decoder家族 · Variants

同样的积木，三种搭法

同一块 Transformer 积木，靠"谁能看见谁"（注意力掩码）的不同，搭出了三个家族：

三种注意力可见性 / Masks
绿格 = 允许注意；空格 = 被遮住。差别只在掩码形状，以及解码器是否额外带一层交叉注意力。
ENCODER-ONLYBERT 一类：双向注意力，每个词能看见左右全部。擅长"理解"——分类、抽取、检索。
DECODER-ONLYGPT 一类：因果掩码（causal mask），每个词只能看见自己及左边，看不到未来。擅长"生成"——逐字续写。
ENCODER-DECODER原版 / T5：编码器双向读完输入，解码器一边因果生成、一边用交叉注意力回看编码器输出。擅长"转换"——翻译、摘要。
11Output Head出口 · Unembed

把向量翻译回一个词

向量爬到塔顶，经最后一次 LayerNorm 后，要变回人话。它乘上反嵌入矩阵（unembedding，形状 768 × 50257），得到词表上每个 token 的原始分数 logits；再过一次 softmax，变成一个概率分布。

一个常见的省参技巧是权重共享（weight tying）：反嵌入矩阵直接复用入口那张嵌入表的转置——进、出共用同一张表。对 the cat sat，塔顶给出的下一个词分布大概是这样：

下一个 token 的概率 / Next-token (示意)
▁on0.34
▁down0.18
▁there0.12
▁up0.09
▁quietly0.06
挑一个词（贪心取最大、或按 temperature 采样），接到句尾，整座塔再从头跑一遍——这就是逐字生成（而把已算过的 K、V 存下来不重算，正是 KV 缓存 那篇的事）。
12The Ledger收支 · Accounting

这座塔，到底花在哪

把账算清。一个 124M 参数的 GPT-2，参数主要分布在三处：进出共用的嵌入表、每层注意力的四个投影（Q/K/V/O）、以及每层的 FFN——其中 FFN 约占每个 Block 参数的三分之二。

它最大的代价是注意力的 O(n²)：序列长一倍，注意力计算量就是四倍——这正是长上下文为什么贵，也是后来有了 KV 缓存、FlashAttention、稀疏注意力等一系列优化的原因。但也正因为整座塔只是"相同积木 + 残差流 + 并行注意力"这么朴素，它才能被一路扩展到千亿参数，成为今天所有大模型的地基。
