原理手记 · 位置编码与 RoPE

注意力分不清 谁先谁后。

自注意力天生是「一袋词」——猫追狗 和 狗追猫 在它眼里一模一样。要让模型懂顺序，得亲手把位置喂进去。从最直白的「把位置加进向量」，到 RoPE 改成「把 Q、K 按位置旋转」——一个旋转，就把绝对位置悄悄变成了相对距离。

\(R(m\theta)\)：query 转了 \(m\theta\)（绝对位置 \(m\)） \(R(n\theta)\)：key 转了 \(n\theta\)（绝对位置 \(n\)） 点积里只剩 \(n-m\)：相对位置
0学习的位置参数
每一层Q/K 都重新旋转
128k+它撑起的上下文
01The Blind Spot · Permutation Invariance

注意力为什么分不清顺序

回忆一次 self-attention：token i 的输出，是它的 query 和所有 key 两两点积、过 softmax、再去加权求和 value——写成 \(\;\text{out}_i=\sum_j \text{softmax}_j(q_i\cdot k_j)\,v_j\)。盯着这条式子看：没有任何一处用到「这个词排第几」。

于是一旦把输入词序打乱，\(q_i,k_j,v_j\) 只是跟着一起搬家，任意一对的分数纹丝不动——这套运算是置换等变（permutation-equivariant）的，天生不含位置。RNN 靠递归一步步走、CNN 靠卷积窗的位置，顺序是结构自带的；Transformer 为了能并行抛弃了递归，顺便也把顺序丢了。只能手动补回来。

交互 · 打乱词序，注意力眼中还是同一袋词
输入顺序
注意力眼中猫追狗＝同一个集合
输入怎么排，上面那行就怎么动；可下面这袋词——以及它们两两之间的注意力分数集合——一字不差。「猫追狗」与「狗追猫」对它是同一道题。位置，必须由我们注入。
02First Instinct · Absolute Encoding

第一感：把位置加进去

最直白的修法——给每个位置一个向量 \(p_{\text{pos}}\)，加到词嵌入上：\(\;x_i=\text{embed}(w_i)+p_{\text{pos}}(i)\)。这样第一层拿到的向量里，就带上了「我在第 \(i\) 位」。两种主流做法：

可学习的绝对编码（BERT、GPT-2）：一张 \([\,L_{\max},\,d\,]\) 的查找表，像普通嵌入一样训出来。简单好用，但有硬天花板——训练到 1024 位，第 1025 位根本没有向量、从没见过，不能外推；还白占参数；而且它编的是绝对序号，可语言常常只在乎相对偏移。

正弦编码（原版 Transformer，无参数）：每一维是不同频率的正弦/余弦——\(\;PE(pos,2i)=\sin\!\big(pos/10000^{2i/d}\big)\)，\(\;PE(pos,2i{+}1)=\cos\!\big(pos/10000^{2i/d}\big)\)。像一排转速按几何级数排开的时钟指针，合起来给每个位置一个独一无二的指纹。妙处在于 \(\sin/\cos\) 的和角公式让 \(pos{+}k\) 的编码是 \(pos\) 的一个固定线性变换，模型理论上能读出相对偏移；又因为是公式，能算到没见过的位置。

正弦位置编码 · 行＝位置 0–15，列＝维度（左快右慢）
−1+1左侧维度高频（相邻行就变色），右侧维度低频（跨很多行才缓缓变化）
两种做法有个共同硬伤：位置只在输入处加一次，立刻和内容缠进同一个向量。点积 \(q_i\cdot k_j\) 里混着「内容×内容、内容×位置、位置×位置」各项，相对位置的信号有，却间接、易被冲淡。我们想要更干净的东西。
可学习查表，简单，但有长度上限、不能外推 正弦无参数、按公式、可外推到更长 共同硬伤只在输入加一次，与内容纠缠
03What We Want · Relative Position

我们真正想要的：相对位置

把目标写死。位置 \(m\) 的 query 与位置 \(n\) 的 key 之间的分数，应当只依赖它俩的内容和相对距离 \(m-n\)，而不分别依赖 \(m\)、\(n\)。「the cat」这对搭配，在句首 \((2,3)\) 和在千字之后 \((102,103)\)，绑定方式应当一样。形式上，我们要一个函数 \(g\)，使得

\[ \langle q_m,\,k_n\rangle = g\big(q,\,k,\;\htmlClass{m-v}{m-n}\big) \]

相对位置流派都在直接追这个目标：Shaw 等人 2018 给注意力加相对位置项；T5 干脆给每个分数按距离桶加一个学到的标量偏置 \(b_{m-n}\)；Transformer-XL 改写了分数展开式。都有效——但都是外挂的加性补丁，往往逐层去学，且不易优雅地外推。

RoPE 的问题更尖锐：能不能不加任何偏置、不学一个位置参数，纯靠变换 \(q\)、\(k\) 自己，就精确拿到这条相对距离性质？

04The Trick · Rotate, Don't Add

核心戏法：旋转，而不是相加

RoPE 的洞见一句话：不「加」位置向量，而是把每个 query / key 按它的位置「旋转」一个角度。神奇之处在于——旋转之后两个向量的点积，只依赖它们旋转角之差，也就是相对位置。

先看二维。把一个二维向量看成平面上的点，旋转角 \(\alpha\) 就是乘上旋转矩阵 \(R(\alpha)=\left(\begin{smallmatrix}\cos\alpha & -\sin\alpha\\ \sin\alpha & \cos\alpha\end{smallmatrix}\right)\)。让 query 在位置 \(m\) 转 \(m\theta\)、key 在位置 \(n\) 转 \(n\theta\)，再做点积，只用两条小学过的性质——\(R(\alpha)^{\top}=R(-\alpha)\) 和 \(R(a)R(b)=R(a{+}b)\)——位置就坍缩了：

四步代数 · 看 m、n 如何坍缩成 n − m
起点分别旋转\(q_m=R(m\theta)\,q\)，\(k_n=R(n\theta)\,k\)——各自带上自己的绝对位置。
展开写出点积\(\;q_m\cdot k_n=\big(R(m\theta)q\big)^{\top}\big(R(n\theta)k\big)=q^{\top}R(m\theta)^{\top}R(n\theta)\,k\)
合并转置即反向、连乘即相加\(\;R(m\theta)^{\top}R(n\theta)=R(-m\theta)R(n\theta)=R\big((n-m)\theta\big)\)
收束只剩相对位置\(\;q_m\cdot k_n=q^{\top}R\big(\htmlClass{m-v}{(n-m)}\theta\big)\,k\)——\(m\)、\(n\) 消失，只留下 \(n-m\)。
对各自施加绝对旋转，点积自动变成相对角的函数。这正是上一节要的 \(g(q,k,\,n-m)\)——精确达成，零新增参数、零加性偏置。
05By the Numbers · A 2D Walkthrough

连数字一起算

取一对维度，固定 \(\theta=1\)（每步转 1 弧度），并把 query、key 都设成单位向量 \(q=k=[1,0]\)（方向相同，方便看）。这时它俩的点积就干净地等于 \(\cos\big((n-m)\theta\big)\)。拖动下面两个滑块改变各自的位置，看夹角和得分怎么变；再点「整体平移」，验证只要间隔不变、分数就不变。

06Going High-Dim · A Ladder of Frequencies

升到高维：一架频率的梯子

真实向量是 \(d\) 维（比如每个注意力头 128 维）。RoPE 把这 \(d\) 维切成 \(d/2\) 个相邻的对，每一对用自己的频率旋转：\(\;\theta_i=10000^{-2i/d}\)，\(i=0,1,\dots,d/2-1\)。第 0 对转得最快（\(\theta_0=1\)），最后一对转得极慢。整体就是一个块对角旋转矩阵：对角线上排着 \(d/2\) 个 \(2\times2\) 小旋转块，第 \(i\) 块转 \(m\theta_i\)。和正弦编码同一架几何频率梯子——只是这回拿来旋转 \(q,k\)，而非加到 \(x\) 上。

为什么要一排频率？高频对转得快，能分辨相邻词的细微先后，但很快绕圈、长距离上就分不清；低频对转得慢，跨越很远仍可区分。合起来，是一把多尺度的尺子——像秒针、分针、时针，各管各的尺度，一起把位置钉死。

频率梯子 · 同一段距离里，高频转好几圈，低频才走一点

而所有对的点积，是一大堆 \(\cos\big((n-m)\theta_i\big)\) 之和。沿着频率梯子叠加之后，会得到一条随距离整体衰减的交互曲线——离得越远的两个词，天然耦合得越弱。这就是 RoPE 著名的长程衰减（long-term decay），一个相当合理的先验。

长程衰减 · 把各频率的 cos 叠起来，随相对距离 |n − m| 增大而回落
07In Practice · The Element-wise Form

实现：不真的去乘那个大矩阵

那个块对角矩阵几乎全是 0，谁也不会真去构造它。真实代码（Llama、GPT-NeoX）用的是逐元素形式：把预存的角度向量 \(\cos(m\Theta)\)、\(\sin(m\Theta)\) 拿来逐元素相乘相加，

\[ \text{RoPE}(x,m)=x\odot\cos(m\Theta)\;+\;\text{rotate\_half}(x)\odot\sin(m\Theta) \]

其中 \(\odot\) 是逐元素乘，\(\text{rotate\_half}\) 把向量配对后取 \([-x_2,\,x_1]\)——正好和 \(\sin\) 相乘，补齐二维旋转的另一半。展开一对就能验证它恰好等于旋转矩阵：

\[ \begin{bmatrix}x_1\cos\theta - x_2\sin\theta\\[2pt] x_1\sin\theta + x_2\cos\theta\end{bmatrix}=R(\theta)\begin{bmatrix}x_1\\ x_2\end{bmatrix}\;\checkmark \]

每个 token 不过两次逐元素乘加，开销可忽略。还有三个关键细节常被忽视：RoPE 只作用在 Q、K 上（绝不碰 V）；它在每一层注意力里、就在 \(QK^{\top}\) 之前施加，而不是在输入处加一次；并且全程没有一个可学习参数，纯是固定的几何。

Q、K只旋转这两个，V 不动
每一层位置在用到它的地方重注入
0 参数固定几何，不学习
≈2 次乘加每 token 的额外开销

为什么 RoPE 赢了

把前面几节的好处收拢成一张清单——这也是它如今几乎成为开源大模型默认位置方案（Llama、Qwen、Mistral、DeepSeek…）的原因。

RoPE 的六条性质
相对内建绝对旋转 ⇒ 相对点积，精确，无须学习位置 作用 Q/K住在注意力计算处，逐层重注入，不被稀释进残差流 长程衰减多频叠加天然削弱远距离耦合，是个好先验 零额外参数不动 V、不占显存、几乎零算力 兼容线性注意力它只是对 q、k 的变换，可塞进核方法 / Performer 一类 利于外推是公式而非学到的表，超长位置仍给出有定义的旋转
09Stretching Context · θ-base, PI, NTK, YaRN

把上下文撑长

RoPE 在今天的分量，正来自长上下文。杠杆就是频率基底那个 10000。每一对的波长是 \(2\pi/\theta_i\)：低频对的波长比训练上下文还长，高频对则短得多。一旦推理时超出训练长度，那些快指针会转到训练时从没见过的角度——模型就崩了。修法，本质上都只是「重新拨一拨角度」。拖动下面的滑块，看位置插值如何把更长的序列压回训练见过的旋转范围。

交互 · 位置插值（PI）：目标越长，刻度越被压回阴影区（训练范围）
目标长度 ×1

绿刻度＝序列里的位置。倍数越大，它们被越使劲地挤进模型训练时见过的旋转区间（阴影）——这就是 PI 用分辨率换范围的代价。
PI位置插值 Position InterpolationChen et al.把每个位置按 \(L_{\text{train}}/L_{\text{new}}\) 缩小（\(m\to m\cdot L_{\text{train}}/L_{\text{new}}\)），让 8k 序列只复用模型见过的 0–2k 旋转范围。便宜，需少量微调，用分辨率换范围。
NTKNTK-aware 缩放调基底不一刀切压所有频率（那会毁掉高频的短程细节），而是增大基底（\(10000\to\) 更大），让低频拉长去覆盖新长度、高频几乎不动——「低频插值、高频外推」，常常不微调也能用。
YaRNYaRN逐频率斜坡只插值需要插的那部分频率，短程频率原样留着，再配一点注意力温度修正——是当前廉价扩展上下文的 SOTA。

这就是为什么你会在 Llama 3 的配置里看到 "rope_theta": 500000、在 HuggingFace 模型里看到 rope_scaling 块：全是在给 RoPE 的旋转速度，配一把更长的尺子。
