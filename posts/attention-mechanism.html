A Field Guide · 注意力机制

注意力， 逐矩阵拆开。

softmax(QKᵀ/√dₖ)·V —— 这条被反复背诵的公式，到底在算什么？我们用一句三个字的话「猫 很 累」，把每一个矩阵乘法、连同每一个数字，一步一步算给你看。

3个词的例子
6步矩阵运算
01The Idea问题

让每个词，看见其他词

一个词向量刚进入网络时，只「认识自己」。可「累」要表达清楚，必须知道是谁累了——它得回头看看「猫」。注意力机制做的，正是让每个词环顾整句，按相关程度给其他词分配权重，再把它们的信息加权汇总进自己。

「累」该把注意力分给谁？
这三个数字（0.55 / 0.12 / 0.33）就是注意力权重。本文要做的，就是把它们一步步算出来——它们不是拍脑袋，而是矩阵运算的结果。
02Input Matrix准备

一句话，变成一个矩阵

每个词先被嵌入成一个向量（这里取维度 d=4，真实模型里是几百上千维），再加上位置信息。三个词堆叠起来，就是输入矩阵 X ∈ ℝ³ˣ⁴——三行（每行一个词）、四列（每列一个特征）。

输入 / X = 词向量堆叠
下面所有运算，都从这张 3×4 的表开始。数字是为了好算而特意挑的，但运算规则与真实模型一模一样。
03Q · K · V投影

三把钥匙：Query、Key、Value

注意力不直接用 X，而是先把它投影成三份。X 分别乘上三个可学习的权重矩阵 W_Q、W_K、W_V，得到 Q（查询）、K（键）、V（值）。这三个权重，正是预训练学到的东西。

三种角色 / The three roles
Query 查询我在找什么？ Key 键我能用什么标识被找到？ Value 值我实际携带的信息。
投影结果 / Q = X·W_Q，K = X·W_K，V = X·W_V
打个比方：每个词都举着一块查询牌 Q（我想找什么）、一块标识牌 K（我是什么）、和一份货物 V（我能给出什么）。接下来，就是拿查询牌去对所有标识牌。
04Scores = QKᵀ打分

打分：Query 与 Key 做点积

每个词的查询牌 q，去和每一个词的标识牌 k 做点积。点积越大，说明两者越「对得上」。所有点积排成一张 n×n 的分数表 S = QKᵀ：第 i 行第 j 列，就是「词 i 该多关注词 j」。

手算 / 「累」看「猫」的分数 = q累 · k猫
点积就是对应位置相乘再相加。3 这个分数，来自 q累 与 k猫 在第 1、第 3 维上的「共振」。
完整分数矩阵 / S = QKᵀ（3×3）
高亮的「累」这一行 [3, 0, 2]：它最想看「猫」(3)，其次看自己(2)，几乎不理「很」(0)。但这还是原始分数，又大又不规整——下一步先把它驯服。
05Scale ÷√dₖ稳定

缩放：除以 √dₖ

维度越高，点积的数值就越容易变得很大；而过大的分数会把下一步的 softmax 推进饱和区，那里梯度几乎为零、模型学不动。所以先统一除以 √dₖ 把方差拉回 ~1。这里 dₖ=4，于是除以 2。

缩放「累」这一行 / S累 ÷ √4
[3, 0, 2] ÷ 2 = [1.5, 0, 1.0]。数值变温和了，但相对大小不变——「猫」依然领先。缩放只稳定数值，不改变偏好。
06Softmax归一

Softmax：把分数变成权重

对每一行做 softmax：先取 eˣ 放大差距，再除以总和归一。出来的一行数字非负、且加起来正好等于 1——这就是注意力权重 α。「累」这一行，终于变成了一个干净的概率分布。

softmax([1.5, 0, 1.0]) → 权重 α
→ 猫0.55
→ 很0.12
→ 累0.33
e¹·⁵=4.48, e⁰=1, e¹·⁰=2.72，三者之和 8.20。各自一除：0.55 / 0.12 / 0.33——正是第 1 站那三个数字的来历。
07Weighted Sum · αV输出

加权求和：按权重取出 Value

最后一步，也是收获的一步：拿刚算出的权重 α，去给每个词的值向量 V 加权求和。o累 = 0.55·v猫 + 0.12·v很 + 0.33·v累。「累」的新向量，从此装进了整句话的信息。

混合 / o累 = Σ αᵢ·vᵢ
因为这里 V 取成了单位向量，输出向量可以被直接读出来：「累」的新表示 = 55% 的猫 + 12% 的很 + 33% 的自己。这，就是注意力干的全部事情。
三个词各自的权重 / 注意力热力图 α
每一行都是一个词的注意力分配（一行和为 1）。颜色越亮，关注越多。把三行叠起来，就是这句话内部的全部「目光流向」。
08Causal Mask约束

因果遮罩：不许偷看未来

上面是双向注意力。但用来生成文字的语言模型有一条铁律：预测某个词时，不能看见它后面的词。做法很巧妙——在 softmax 之前，把分数矩阵的上三角全部设成 −∞。这样 eˣ 后它们都趋近于 0，未来位置自然分不到任何权重。

遮罩后的分数 / 上三角 = −∞
于是「猫」（第 1 个词）只能看自己；「很」能看「猫、很」；只有「累」（最后一个）能纵览全句。把这块下三角，叫做因果遮罩 causal mask。
09Multi-Head扩展

多头：并行的多重视角

一次注意力只能捕捉一种关系。于是干脆并行跑多个——每个「头」有自己独立的 W_Q/K/V，各自投影到 dₖ = d/h 维。不同的头会自发分工：有的盯语法，有的追指代，有的看位置。算完把所有头拼接起来，再过一个输出投影 W_O。

一个典型配置 / Typical config
512模型维度 d
8注意力头 h
64每头维度 dₖ
×N逐层重复
八个头，各看各的 / 8 heads, 8 patterns
每个小方阵是一个头的注意力图样——彼此截然不同。正是这种多样性，让模型能同时兼顾语法、指代、远近等多种关系。
10The Formula收束

公式，归位

它在 Transformer 里的位置 / In the block
1

注意力负责「让词互相看见」，前馈网络负责「各自深入思考」。两者交替几十层，语言的理解便层层浮现。
