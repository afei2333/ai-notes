A Field Guide · 大语言模型的一生

万亿字节， 凝成一句回答。

从散落于互联网的原始文本，到此刻屏幕上流淌而出的文字——一个大语言模型要经历训练、对齐与推理的漫长旅程。让我们一阶段、一阶段地走完它。

01 Data Collection 训练阶段

汇聚世界的文本

模型的认知，始于阅读。工程师从公开网页、书籍、代码仓库、百科与论坛中收集海量文本，构成原始语料。规模常达数万亿 token——相当于一个人不眠不休读上数千年。

02Cleaning & Dedup训练阶段

去芜存菁

原始语料噪声极多。系统过滤垃圾内容、去除重复、剔除隐私与有害信息，并按质量对文本加权。一条朴素的真理在此显现：数据质量，决定模型的上限。

质量过滤 / 原始 → 保留
03Tokenization训练阶段

把文字变成数字

模型不认识文字，只认识数字。分词器把文本切成 token——介于字与词之间的最小单元，再为每个 token 分配一个整数 ID。自此，语言成为一串可计算的序列。

04Pre-training训练阶段

预测下一个词

这是整段旅程中最昂贵、最核心的一步。在数千张 GPU 上，模型日夜不停地重复一件朴素到极致的事：给定一段上文，预测下一个 token。没有人雇佣标注员——文本自己就是答案，被遮住的下一个词，就是标准答案。这种「自监督」方式，让模型得以吞下几乎整个互联网。

关键在于：模型一开始什么都不会，预测纯属乱猜。但每猜一次，系统就量化它错得有多离谱，再据此把数十亿个参数各自向「更对一点」的方向挪动一丝。亿万次这样的微调累积起来，语法、事实、推理与世界常识便被悄然压缩进权重之中。

训练目标 / 自监督 · 完形填空
人工智能正在改变 ？ ← 模型要猜的下一个 token
"世界"0.71
"生活"0.18
"一切"0.06
真实文本里的下一个词是 「世界」。模型给它的概率越高，损失越小；猜偏了，损失就大——这个差距，驱动着整场学习。
训练循环 / 每一步迭代都做这四件事
1
前向传播Forward把上文送进网络，经过几十层 Transformer，输出对下一个 token 的概率分布。
2
计算损失Cross-Entropy Loss用交叉熵衡量「预测分布」与「真实答案」的差距，得到一个数字——错得越多，它越大。
3
反向传播Backprop沿网络从后往前回溯，算出每一个参数对这次误差「负有多少责任」，即梯度。
4
梯度下降Optimizer优化器（如 AdamW）按梯度方向，把每个参数都微调一点点，让下次预测更准。
四步合为一次 step。成千上万条序列打包成一个 batch 同时计算，再重复 亿万次——这就是预训练的全部。

05Supervised Fine-Tuning训练阶段

学会听懂指令

预训练后的模型博学却不善对话。研究者用人工编写的「指令—回答」示范继续训练，让它学会理解你的意图，并以有条理、对话式的方式回应。

示范数据 / Instruction → Response
指令

用一句话解释什么是黑洞。

回答

黑洞是引力极强的天体，连光都无法从中逃逸。

06RLHF · Alignment训练阶段

向人类的偏好对齐

人们对模型的多个候选回答排序打分，由此训练出一个「奖励模型」；再用强化学习优化策略，让回答更有用、诚实、无害。这一步，赋予模型以分寸感。

候选回答排序 / Reward Ranking
+9.2耐心、准确、给出可执行的步骤★ 偏好
+5.1正确但略显笼统
−3.4含糊、回避问题
07Deployment训练阶段

部署到推理集群

训练好的权重被部署到推理服务器。通过量化压缩、KV 缓存与批处理调度，模型得以低延迟地同时回应海量并发请求。它，终于准备好与你对话。

推理集群 / Inference Cluster
08Your Prompt用户阶段

你，输入提示词

你敲下一句话。它会和系统提示词、历史对话一起，被同样地分词、拼接，组成模型这一轮能「看见」的全部内容——也就是上下文窗口。

09Attention用户阶段

注意力，读懂语义

模型读入全部上下文 token。在每一层里，注意力机制反复权衡每个词与其他词的关系——哪些该被关注，哪些可被忽略——从而构建出对你这句话的内部理解。

注意力权重 / Attention Map
10Autoregressive Decoding用户阶段

逐字生成回答

模型一次只预测一个 token：生成它、把它接回输入、再预测下一个，如此循环。采样策略（温度、Top-p）在确定性与创造力之间悄悄取舍。

回答，浮现于眼前

生成的 token 被还原成文字，流式呈现在你面前。一句轻描淡写的提问，历经万亿参数的运算，化作此刻这段回答。
