写不出 「好答案」的损失。

01The Gap · Beyond Imitation

对齐要补的，是 SFT 补不上的那一段

先把三步流水线摆清。预训练在整个互联网上做 next-token 预测，得到一个「什么都能往下接」的模型——但它续写的是互联网，不是一个好助手。SFT（监督微调 / 指令微调）用人写的 (指令, 理想回答) 对，做和预训练一模一样的交叉熵，只是把数据换成示范，于是模型学会了「被提问时要回答问题」的格式与口吻。

但 SFT 有一道硬天花板：它只有正样本，只会模仿示范，且无法超过示范者；它从没被告诉「什么是差的回答」。可我们想要的「更有用」是一个偏好排序，不是一段能照抄的目标序列。一句话钉死区别——SFT 最大化「示范回答」的似然；对齐则最大化「人类偏好下的期望奖励」，后者第一次有了「A 比 B 好」的相对信号。

全景 · 同一份偏好数据，通向两条对齐路径
两条路同源——都从「人对回答的两两比较」出发。上面一条先把偏好炼成一个 reward model，再用 PPO 在线追它（经典 RLHF）；下面一条（DPO）跳过这一切，直接在偏好对上训一个损失。本文走完它们，再讲取舍。
预训练学「世界长什么样」——海量文本上的自监督 SFT学「助手该怎么答」——模仿示范，只有正例 对齐学「人更喜欢哪一个」——相对偏好，有正有负
02Reward Model · Bradley-Terry

Reward Model：把「我更喜欢」压成一个分数

第一步，把人心里那把模糊的尺子固化成一个可计算的函数。做法：对同一个 prompt 让模型采样出多个回答，请人（或更强的模型）两两比较，得到一堆三元组 (x, y_w, y_l)——同一问题下，赢家 \(y_w\) 与输家 \(y_l\)。然后拿 SFT 模型，把它最后的语言模型头换成一个标量输出头：读入「prompt + 回答」，吐出一个分数 \(r(x,y)\)。

怎么训这个分数？用 Bradley-Terry 模型——一个把「成对比较」接上概率的经典框架。它假设「人选 A 而非 B」的概率，由两者分数之差经 sigmoid 给出：

\[ P(y_w \succ y_l \mid x) = \sigma\big(r(x,y_w) - r(x,y_l)\big),\qquad \sigma(z)=\frac{1}{1+e^{-z}} \]

于是 reward model 的损失就是这件事的负对数似然——把赢家分数推高、输家压低：

\[ \mathcal{L}_{\text{RM}} = -\,\mathbb{E}_{(x,y_w,y_l)}\Big[\log\sigma\big(r(x,y_w)-r(x,y_l)\big)\Big] \]

关键认知：它只在乎两者之差，不在乎绝对值——整张分数表加减一个常数，损失分毫不变。所以 reward model 学的是相对偏好，它的绝对刻度没有意义。拖动下面两个滑块给那条祝福的两个回答打分，看「差」如何决定模型认为「人有多大概率选赢家」，以及当下这条样本的损失。

交互 · Bradley-Terry：分数差 → 选择概率 → 损失
赢家 r(y_w)=2.3
输家 r(y_l)=−0.5
例：祝福 A 打 2.3、敷衍的 B 打 −0.5，差 2.8 → \(\sigma(2.8)\approx0.94\)，模型认定人有 94% 会选 A，损失只剩 \(-\log0.94\approx0.06\)。把两个分数拉到一样，概率回到 50%、损失升到 \(\log 2\approx0.69\)——它对自己的判断毫无把握。

埋一颗雷：这个 reward model 是一个被训练出来的、有限的代理（proxy），不是真的人类偏好。它在没见过的回答上会出错、有自己的怪癖。下一步的 PPO，会拼命去钻它的空子。

03PPO · Reward-Chasing with a Leash

PPO：用强化学习去追这个分数

有了能打分的 reward model，对齐就被改写成一道强化学习题。把语言模型看成策略 \(\pi_\theta\)：状态是「已经生成的前缀」，动作是「下一个 token」，一整条回答采样完，由 reward model 在末尾给一个标量奖励。目标——调 \(\theta\) 让期望奖励最大，但要拴一根绳：

\[ \max_\theta\ \mathbb{E}_{x\sim D,\,y\sim\pi_\theta(\cdot\mid x)}\big[\,r(x,y)\,\big]\;-\;\beta\,\mathrm{KL}\!\big(\pi_\theta(y\mid x)\,\|\,\pi_{\text{ref}}(y\mid x)\big) \]

那根绳就是 KL 惩罚。不拴的话，策略会漂去专门讨好 reward model——生成人根本看不懂、但 RM 偏偏给高分的怪话（这就是 reward hacking / 奖励黑客，下一节细说）。KL 把当前策略拽在 SFT 模型（reference \(\pi_{\text{ref}}\)）附近，别跑太远；\(\beta\) 控制松紧。实现上，这条 KL 通常被摊进每一步的 token 奖励里：

\[ r_t = -\,\beta\big(\log\pi_\theta(a_t\mid s_t)-\log\pi_{\text{ref}}(a_t\mid s_t)\big)\;+\;\underbrace{r(x,y)}_{\text{仅在末端}} \]

为什么不能直接对奖励梯度上升？因为奖励来自「采样 + 打分」这条不可导的链路，只能用 policy gradient（REINFORCE 一类）去估计梯度，而这种估计的方差极大。PPO（Proximal Policy Optimization，近端策略优化）就是让它稳下来的那套方法，两件法宝：

① advantage（优势）+ GAE。不直接用「拿到多少奖励」，而用「比预期好多少」。引入一个 value model（价值网络 / critic） 估计基线 \(V(s)\)，advantage 衡量某一步动作超出基线多少；GAE 用 \(\gamma,\lambda\) 在偏差与方差间权衡，把末端奖励平滑地回传到每个 token：

\[ \hat A_t=\sum_{l\ge0}(\gamma\lambda)^l\,\delta_{t+l},\qquad \delta_t=r_t+\gamma V(s_{t+1})-V(s_t) \]

② clip（裁剪）。用重要性采样比 \(\rho_t=\pi_\theta/\pi_{\text{old}}\) 复用旧样本，再把它剪到 \([1-\epsilon,\,1+\epsilon]\)，让每步只许策略相对旧策略改一点点——这就是 "Proximal（近端）" 的字面意思，防止一次更新步子太大、把策略带崩：

\[ L^{\text{CLIP}}(\theta)=\mathbb{E}_t\Big[\min\!\big(\rho_t\hat A_t,\ \mathrm{clip}(\rho_t,\,1-\epsilon,\,1+\epsilon)\,\hat A_t\big)\Big] \]
PPO 的 clip · 当 advantage > 0，奖励再诱人，单步放大也被锁在 1+ε
设定旧策略给这个 token 概率 0.10新策略很想把它抬到 0.16 —— 重要性比 \(\rho=0.16/0.10=1.6\)，而 advantage 为正（这步比平均好）。
裁剪取 ε = 0.2clip 到 1.2\(\rho=1.6\) 越出 \([0.8,1.2]\)，被剪回 \(1.2\)。
结果目标按 min(1.6·A, 1.2·A) = 1.2·A 计一步最多把这个 token 的概率往上推「相当于 1.2 倍」那么多，多出来的冲动被剪掉——下一轮重新采样、再走一小步。稳，但慢。
04The Cost · Four Models & Reward Hacking

PPO 为什么又重又难

把上一节的零件数一遍，就明白 PPO 的代价了。训练时，显存里同时站着四个模型——这是它重、贵、难调的根源：

Policy在训 · 正在更新的策略
Reference冻结 · 算 KL 拴绳的锚
Reward冻结 · 给回答打分
Value在训 · critic 估基线

四个模型还不是最痛的。最痛的是：你优化的从来不是「人类偏好」，而是它的代理 reward model。一旦策略获得自由，它会像找漏洞一样去钻 RM 的脆弱处——这就是 reward hacking。典型现场：RM 的训练数据里碰巧「长答案常被偏好」，PPO 就把每个回答灌水到老长；或者发现 RM 偏爱「希望这对你有帮助！」这类客套、偏爱满屏 markdown，就疯狂堆砌。回答的 RM 分数一路飙升，对人的真实有用度却在下跌。

这正是 Goodhart 定律：当一个指标成为优化目标，它就不再是个好指标。KL 拴绳能拖慢这种漂移，却治不了本——因为代理本身有缺陷。

Goodhart · 越使劲优化代理，与真目标越背离
reward model 分数（代理）：一路走高 真实人类偏好：先升，后被反噬而回落
两条线在前半程同向，于是早期 RLHF 确实在变好；越过那个背离点，模型开始为了刷分而牺牲真有用度。何时停、KL 多紧、RM 多准——PPO 的大半调参功夫，都花在这道平衡上。
05DPO · Solving the RL Away

DPO：把强化学习这步直接解掉

能不能不要 reward model、不要采样 rollout、不要 PPO，直接在偏好对上训？DPO（Direct Preference Optimization，直接偏好优化）的答案是能——而且只用四步代数。诀窍在于：上一节那个「最大期望奖励 − KL」的目标，其实有一个闭式最优解。

四步推导 · reward model 如何被「解」进策略自身
起点RLHF 的目标\( \max_\theta\ \mathbb{E}_{y\sim\pi_\theta}[\,r(x,y)\,]-\beta\,\mathrm{KL}(\pi_\theta\,\|\,\pi_{\text{ref}}) \)——奖励要高，但别离 reference 太远。
闭式解这目标有解析最优策略\( \pi^{*}(y\mid x)=\tfrac{1}{Z(x)}\,\pi_{\text{ref}}(y\mid x)\,e^{\,r(x,y)/\beta} \)——在 reference 上「按奖励的指数」重新加权；\(Z(x)=\sum_y\pi_{\text{ref}}(y\mid x)e^{r/\beta}\) 是只对 \(x\) 求和的配分函数（归一化常数）。
反解把奖励用策略写回来对上式取对数、移项：\( r(x,y)=\beta\log\tfrac{\pi^{*}(y\mid x)}{\pi_{\text{ref}}(y\mid x)}+\beta\log Z(x) \)——任何奖励，都能这样用「最优策略相对 reference 的对数概率」表示。
抵消代回 Bradley-Terry，Z(x) 消失偏好只看奖励之差，而 \(\beta\log Z(x)\) 只依赖 \(x\)、与回答无关，在 \(r(x,y_w)-r(x,y_l)\) 里 \(\htmlClass{m-v}{\text{整项抵消}}\)。把那个解不出来的归一化常数，干净地约掉了。
于是「人选赢家」的概率，只剩下策略自己的对数概率比值（把最优 \(\pi^{*}\) 换成待训的 \(\pi_\theta\)）——reward model 被彻底解进了策略里。

把它写成最大似然，就是 DPO 的损失——一个能直接反向传播的监督式分类损失：

\[ \mathcal{L}_{\text{DPO}}=-\,\mathbb{E}_{(x,y_w,y_l)}\Big[\log\sigma\Big(\beta\log\tfrac{\pi_\theta(y_w\mid x)}{\pi_{\text{ref}}(y_w\mid x)}-\beta\log\tfrac{\pi_\theta(y_l\mid x)}{\pi_{\text{ref}}(y_l\mid x)}\Big)\Big] \]

翻成人话：语言模型自己，在「相对 reference 的对数概率」这个尺度上，就充当了它自己的 reward model。不再单独训 RM、不再采样 rollout、不再做 RL。训练时显存里只剩 两个模型——在训的 policy 和冻结的 reference。便宜、稳定，几乎像 SFT 一样好训。

06By the Numbers · DPO's Implicit Reward

DPO 的一步更新，连数字一起算

DPO 损失里那一坨 \(\beta\log\tfrac{\pi_\theta}{\pi_{\text{ref}}}\) 有个名字——隐式奖励（implicit reward）。它就是模型悄悄给每个回答打的分：

\[ \hat r_\theta(x,y)=\beta\log\frac{\pi_\theta(y\mid x)}{\pi_{\text{ref}}(y\mid x)} \]

拿那条祝福算一遍。取 \(\beta=0.1\)，把回答的整句对数概率（各 token 之和）写出来——赢家 \(y_w\)（温暖具体那条）、输家 \(y_l\)（敷衍套话那条）在 reference 与当前策略下分别是：

log π_ref(y_w)
−20.0
log π_θ(y_w)
−19.0
Δ_w = 差
+1.0
log π_ref(y_l)
−18.0
log π_θ(y_l)
−18.5
Δ_l = 差
−0.5

隐式奖励之差就是 \(\beta(\Delta_w-\Delta_l)=0.1\times\big(1.0-(-0.5)\big)=\mathbf{0.15}\)；代进 \(\sigma\) 得 \(\sigma(0.15)\approx\mathbf{0.54}\)，这条样本的损失 \(-\log0.54\approx\mathbf{0.62}\)。读数：模型当前只略微偏好赢家，离「确信」还很远，所以损失不小、还要继续学。

这一步梯度会做什么？它抬高赢家、压低输家的相对对数概率，而且按「当前错得多严重」自动加权：

\[ \nabla_\theta\mathcal{L}_{\text{DPO}}=-\,\beta\,\underbrace{\sigma\big(\hat r_\theta(x,y_l)-\hat r_\theta(x,y_w)\big)}_{\text{错判得越狠，权重越接近 1}}\Big[\nabla_\theta\log\pi_\theta(y_w\mid x)-\nabla_\theta\log\pi_\theta(y_l\mid x)\Big] \]

当模型反而给输家更高的隐式奖励（排序错了），那个 \(\sigma(\cdot)\) 权重趋近 1，更新最猛；当它已把赢家排在前头，权重趋近 0，几乎不动——天然的「难例优先」。这正是 DPO 把整套 PPO（reward model + 采样 + clip 更新）压成一个损失后，依然奏效的内在道理。

⚠ 但要当心两个失败模式：只要「差」在涨，赢家与输家的绝对概率可能一起掉，把概率质量挤到分布外的怪地方；\(\beta\) 太小则会过度偏离 reference。DPO 稳，不等于没有坑。
07The Trade-off · Online vs Offline

取舍：在线 vs 离线，才是分水岭

DPO 这么省，为什么 PPO 没被淘汰？因为两者最深的差别不在「省不省」，而在信号怎么来。PPO 是在线（on-policy）的：每一步都用当前策略采新回答，reward model 能对「模型此刻真会生成的分布」打分，于是能持续探索、走出偏好数据没覆盖的新区域。DPO 是离线（off-policy）的：只在一份固定的偏好集上学，数据没覆盖的回答，它拿不到任何信号——质量上限被数据集框住。

DPO 还对偏好数据的分布敏感：若偏好对里的回答不来自（接近）被训模型自己的分布，效果会打折——实践中常先用待训模型自己采样、再标注来缓解，让离线数据「贴近在线」。

所以怎么选？资源有限、要快速可复现 → DPO，它如今是多数开源对齐的默认起点。追求最高上限、有工程能力、需要在线探索（尤其数学、代码这类有可验证答案的任务）→ on-policy RL。现实里的前沿做法常是混合：先用 DPO 做便宜的第一轮对齐，再用在线 RL 精修。

08The Aftermath · RLAIF, GRPO, RLVR

余波：这条线正在往哪走

RM / PPO / DPO 之后，对齐这条线还在快速演化，但拨的都是同三个旋钮——信号从哪来、要不要 critic、代理多容易被钻空子。几个绕不开的名字：

GRPO（Group Relative Policy Optimization，组相对策略优化）把 PPO 的 value model 砍掉了。它怎么估 advantage？对同一个 prompt 一次采样一组 \(G\) 个回答，每个拿到奖励 \(r_i\)，直接用这组的均值和标准差做归一化——「比同组平均好多少」，就是 advantage：

\[ \hat A_i=\frac{r_i-\operatorname{mean}(r_1,\dots,r_G)}{\operatorname{std}(r_1,\dots,r_G)} \]
GRPO · 用一组样本的相对名次当基线，省掉 value model
同一道题采 \(G\) 个答案，画一条组内均值线；高于均值的样本得正 advantage（被强化），低于的得负（被抑制）。不再需要单独的 critic——内存里又少一个模型，估计也更稳。DeepSeek 在数学 / 推理模型上用的就是它。

RLVR（RL with Verifiable Rewards，可验证奖励的 RL）换掉了「奖励从哪来」：不再用一个学出来的 reward model，而用一个可验证的检查器——数学题对答案、代码过单元测试。于是那层「会被 hack 的 RM」整个消失了（虽然验证器本身仍可能被钻，比如蒙对答案、写退化代码骗测试）。把模糊的人类偏好换成客观真值，正是 o1 / R1 这批推理模型爆发的关键。

RLAIF用 AI 代替人来标偏好含 Constitutional AI让更强的模型按一套写明的原则去比较回答，产出偏好标注，绕开人工标注的瓶颈与成本。信号来源：人 → AI。
GRPO组内相对优势，去掉 critic更轻的 on-policy RL用一组样本的相对名次当基线，省下 value model；让在线 RL 在推理任务上更省、更稳。
RLVR奖励来自可验证的对错数学 / 代码检查器代替学出来的 RM，根除「刷 RM 分」这层 hack；是推理模型的主引擎。
KTO / IPODPO 的变体放宽数据 / 修过拟合KTO 只需「好/坏」二元标签、不必成对；IPO 修正 DPO 在偏好近乎确定时的过拟合。

把它们串起来，会发现对齐的核心张力始终如一——我们没有「好答案」的损失函数，只能用一个会被钻空子的代理，去逼近人的偏好。RM + PPO、DPO、可验证奖励，不过是在「信号从哪来、优化得多激进、代理多容易被 hack」之间，做了不同的取舍。

End of Field Note

没有「好答案」的损失函数——对齐能做的，是教模型学会人类的「更喜欢」，再小心别让它学会钻空子。
